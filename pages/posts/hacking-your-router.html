Hacking your router in 1 minute

Hello everybody! I hope everyone is well. Today in this post will share with you guys a easy method to search for vulnerabilities in your router with just one tool!

ü§î Vulnerabilities in routers

It's not a new for everyone that routers has been a lot explored in the internet wild, you can easily find a bunch of articles on the internet written by security researchers Who have found critical vulnerabilities that can allow a attacker to bypass the authentication, get informations from the victim or gain access in a device.

The main reason why routers are exploited is to create big botnets which are going to be used to DDoS websites or people around the world. (Obviously, this not only the unique reason)

My goal in this article is to help people to find vulnerabilities on his router and try to fix or report to their ISP.

üß∞ Setuping the tool

The tool we are going to use is called "Routersploit" and can be find here https://github.com/threat9/routersploit. One important thing is you need to be on Linux to install this tool or use a Docker container in your Windows machine

You can follow the documentation created by the project maintainers and install the tool in your machine. My current OS is Kali Linux so I'll follow what is on the documentation.

üîç Searching for vulnerabilities

Now we are in the cool part, I executed Routersploit with python3 rsf.py and we can see a beautiful menu showing for us a lot of things

We have 134 exploits, 4 scanners, 171 credentials, 4 generic vulnerabilities, 32 payloads and 0 enconders! As you can see a big arsenal to try to find vulnerabilities in us router.

The easiest way to scan for vulnerabilities with Routersploit is using the scanner autopwn which will run all exploits,creds and generic modules for us

First of all, we need to get our IP address, we can do that using the command ifconfig and get the eth0 address

After this we can back to Routersploit, setup everything and run the scanner

use scanners/autopwn
set target your.ip.address
run

Scanner will start and run all available modules trying to find any vulnerability. What we need to do is just wait for the scan finish and hope that nothing will be found. Why? Safety comes first baby

üìù Conclusion

As per the image above, there's not any vulnerability in my current router and I'm a bit more secure than the other people who use old routers or with known-vulnerabilities

If Routersploit found any vulnerability in your router I highly recommend you changing your router for one more new or secure. Keep that on your mind: A hacker need only one apple to get all the forest
